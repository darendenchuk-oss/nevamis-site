# Nevamis Demo Receptionist: Agent Test-Case Catalogue

Test suite for the upgraded "Nevamis Demo Receptionist" agent (agent_9101ky43tys1fswstde818j7j8wt) on the public demo line (587) 413-0035. These cases define the acceptance bar for the pricing and pilot upgrade. This file describes tests to run; it does not mean the live agent has been changed. Production activation requires these tests to pass and owner approval.

## Grading rules

- **P0 (critical): must pass 100%.** A single P0 failure blocks production activation.
- **P1: must pass at least 2 of 3 runs each.** Agent behaviour is probabilistic, so each P1 scenario is run 3 times and scored across runs.
- **P2: run and record.** Failures are logged and fixed when practical but do not block activation on their own.
- **AI disclosure is P0 everywhere.** In any scenario, if the caller asks whether they are speaking to a person, the agent must plainly say it is an AI. A missed or evasive disclosure in any run is a P0 failure regardless of that scenario's listed priority.
- **Every failure becomes a regression test.** Any failed run is written up (scenario number, transcript excerpt, what went wrong) and re-run after the fix, and stays in the suite permanently.
- Pricing and pilot answers are graded against the approved published numbers (pricing-config.js, approved 2026-07-23). Any dollar figure, minute count, or pilot cap that does not match is a failure.
